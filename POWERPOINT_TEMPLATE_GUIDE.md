
# PowerPoint 네이티브 템플릿 가이드 — 슬라이드 마스터로 틀 고정하기 `DESIGN_SYSTEM.md`는 HTML/CSS로 틀을 고정하는 방법입니다. 실제 `.pptx` 파일에서 같은 원칙 ("틀은 고정, 내용만 채운다")을 구현하려면, 구분선·대외비 배지·footer 같은 장식·구조 요소를 일반 슬라이드가 아니라 **슬라이드 마스터/레이아웃**에 두고, 콘텐츠는 **placeholder**로만 채우게 합니다. ## 왜 마스터/레이아웃에 두는가 - 일반 슬라이드에 그려둔 도형은 콘텐츠를 다시 생성할 때(사람이든 AI든) 실수로 지워지거나 옮겨질 수 있습니다. - 마스터/레이아웃에 있는 도형은 "배경 그래픽"으로 취급되어 콘텐츠 편집 흐름에서 잘 건드려지지 않습니다. - 콘텐츠 생성 도구는 보통 "이 슬라이드의 placeholder에 무엇을 넣을지"만 판단하고 마스터 자체를 다시 그리지 않습니다 — "틀은 그대로, 내용만 채운다"는 워크플로우와 정확히 맞아떨어집니다. ## 구성 방법 (요약) 1. PowerPoint → 보기 → 슬라이드 마스터. 2. 마스터에 공통 요소(폰트, 색 테마, 배경)를 지정합니다. 3. 필요한 만큼 **사용자 지정 레이아웃**을 만듭니다(예: "2단 본문", "3단 본문", "전체활용"). 레이아웃마다: - Title Placeholder 1개(제목) - Text Placeholder 1~2개(거버닝/부연) - Content Placeholder 2~3개(좌우 패널 본문 — 표·텍스트를 자유롭게 넣을 수 있는 기본 콘텐츠 placeholder) 4. **레이아웃(슬라이드가 아니라 레이아웃 자체)에 고정 도형을 그립니다**: 제목 아래 구분선, 우상단 배지, 하단 footer 선·페이지 번호·조직정보. placeholder가 아닌 일반 도형으로 두면 콘텐츠 생성 시 거의 건드려지지 않습니다. 5. 마스터 보기를 닫고, 실제 슬라이드에서 홈 → 새 슬라이드 → 방금 만든 레이아웃을 선택합니다. Placeholder만 빈 상태로 보이고, 구분선·배지·footer는 이미 떠 있는 상태로 시작합니다. 6. `.potx`(PowerPoint 서식 파일)로 저장하면 재사용 가능한 템플릿이 됩니다. ## MS Copilot 사용 시 주의점 - **"이미 템플릿이 적용된 슬라이드의 내용을 채우거나 다시 써 달라"는 요청은 레이아웃이 유지될 가능성이 높습니다.** "이 자료로 프레젠테이션을 새로 만들어줘"라는 요청은 Copilot Designer가 자체 테마를 적용해 틀을 덮어쓸 수 있습니다 — 이 경우 결과물에 템플릿을 다시 적용해야 할 수 있습니다. - 조직의 Copilot 환경에 회사 브랜드 템플릿이 기본으로 등록돼 있으면 이 문제가 줄어듭니다 (테넌트 관리자 설정 영역이라 별도 확인 필요). - **완전한 잠금은 아닙니다.** HTML+CSS(이 저장소의 `DESIGN_SYSTEM.md`)는 브라우저가 그 CSS를 그대로 렌더링하는 결정론적 방식인 반면, PowerPoint+Copilot은 AI가 레이아웃을 다시 고를 여지가 남아 있습니다. 절대 깨지지 않는 결과가 필요하다면, "고정 템플릿의 지정된 자리에 텍스트만 결정론적으로 주입"하는 코드 기반 파이프라인이 더 안전하고, Copilot은 초안 문구 작성에만 쓰는 방식으로 역할을 나누는 것을 권합니다.
