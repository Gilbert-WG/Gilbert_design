# AI-USAGE — 외부 AI(예: MS Copilot)가 이 저장소를 쓰는 규칙

이 문서는 외부 AI 어시스턴트가 이 저장소를 참고해 슬라이드·문서를 만들 때 **무엇을 고정하고, 무엇을 자율로 채우고, 무엇을 넣지 말아야 하는지**를 정의합니다.

## 1. 고정 영역 (바꾸지 말 것)

- **CSS**: `design-guide/css/slide-base.css` — 한 글자도 바꾸지 않고 `<head><style>`에 한 블록으로 복사.
- **동작 스크립트**: `design-guide/script/slide-nav.js` — 한 글자도 바꾸지 않고 `</body>` 직전 `<script>` 하나로 복사.
- **DOM 골격·클래스명**: `slide-html-guide.md` §3의 중첩 구조·클래스명·태그.
- **화면 규격**: 1280×720 고정, 좌우 2단 패널 구조, 색·폰트·여백.

이 영역을 바꾸면 화면 안에서 잘림·넘침이 생기거나 동작이 깨집니다.

## 2. 자율 영역 (자유롭게 채울 것)

- **본문 내용**: 제목·거버닝·부연·패널 텍스트·표 데이터 — 주제에 맞게 자유롭게 구성.
- **구성·논리 전개**: 어떤 관점으로 나눌지, 표를 쓸지, 몇 면으로 나눌지.
- **표현·문장**: `tone-and-voice/writing-style.md`의 말투를 지키는 선에서 자유.

즉, **"틀은 고정, 알맹이는 자유"**가 원칙입니다.

## 3. 넣지 말아야 할 것 (중요)

- **실재하는 특정 회사·조직·팀·거점의 실명과 내부 데이터**를 지어내지 말 것. 모르는 값·내부 수치는 `[확인 필요]`로 표기(수치 날조 금지).
- 이 저장소 자체에 **기밀·내부 정보를 커밋하지 말 것.** 예시는 항상 가상·범용으로.
- 외부 리소스(CDN·웹폰트·원격 이미지·SVG·Base64) 삽입 금지 — self-contained HTML 유지.
- 슬라이드에 **지침 작성자·버전·"made by" 표기**를 넣지 말 것.

## 4. 최소 시작 절차

1. `design-guide/templates/single-slide.html`(또는 `deck-multi.html`) 복사.
2. 대괄호 플레이스홀더(`[키워드]`, `[항목]`, `[값]` 등)를 실제 내용으로 교체.
3. `slide-html-guide.md` §6 자가 검증 항목으로 잘림·논리·표기 점검.
4. `slide-html-guide.md` §7대로 HTML 뒤에 검토 의견 3종(보완점·추가 필요 정보·제안사항)을 붙여 응답.

## 5. PPT로 옮길 때

- 동일한 시각 규격(1280×720, 좌우 2단, 색·폰트, 개조식 말투)을 슬라이드 마스터로 재현.
- 텍스트 계층: 제목 → 거버닝(`□`) → 부연(`-`) → 패널 제목(`【 】`) → `□`/`-`/`·` 글머리.
- 표는 실제 표 개체로(이미지·텍스트 흉내 금지).
