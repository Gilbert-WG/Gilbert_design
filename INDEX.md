# 디자인 레지스트리 이 파일은 "지금 이 저장소에 어떤 디자인이 등록돼 있는지"를 알려주는 목록입니다. 사람이 아니라 **에이전트가 가장 먼저 읽는 진입점**입니다 — 새 디자인이 추가돼도 이 파일의 항목만 늘어날 뿐, 기존 항목의 경로·의미는 바뀌지 않습니다(기존 자동화가 깨지지 않도록 보장). ## 사용 규칙 (에이전트용) 1. 문서를 만들기 전에 이 파일을 항상 먼저 읽는다. 2. 사용자가 디자인 id를 지정하지 않았으면 `default_design` 값을 쓴다. 3. 선택한 디자인 항목의 `design` 경로(틀)와 `content` 경로(내용 가이드)를 모두 읽는다 — `design`만 읽고 `content`를 건너뛰지 않는다. 4. 아래 목록에 없는 id를 사용자가 요청하면, 지어내지 말고 목록에 없다고 답한다. ``` default_design: default ``` ## 등록된 디자인 ### `default` — 고정형 16:9 임원보고 슬라이드 (좌우 2단, 표/차트/플로우 지원) - 틀(디자인, 불변): `designs/default/DESIGN_SYSTEM.md` - 내용 가이드(큰 틀만, 표현은 자율): `designs/default/CONTENT_GUIDE.md` - 실제 PowerPoint(.pptx)로 만들 때: `POWERPOINT_TEMPLATE_GUIDE.md`(모든 디자인 공통) <!-- 새 디자인을 추가할 때 이 형식으로 항목을 이어 붙인다 (예시, 실제 등록 전까지는 존재하지 않음): ### `<새 id>` — <한 줄 설명> - 틀: designs/<새 id>/DESIGN_SYSTEM.md - 내용 가이드: designs/<새 id>/CONTENT_GUIDE.md -->
